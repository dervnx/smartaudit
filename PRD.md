## 1. 产品概述

### 1.1 产品背景

智核 SmartAudit 是一款面向企业级用户的智能审核系统，旨在为具有数据审核、校验、核查需求的机构提供一站式解决方案。系统支持灵活配置审核规则，对接第三方权威数据源，实现自动化审核与人工复核相结合的高效工作模式。

### 1.2 产品定位

- **目标用户**：需要对学生信息、职业资格、身份认证、学历学位等数据进行审核校验的企业和机构
- **核心价值**：提升审核效率、降低人工误差、追溯审核轨迹、统计工作绩效
- **应用场景**：教育机构的人事审核、HR的人力资源校验、金融机构的资质审查、政府部门的数据核查

### 1.3 产品组成

本产品由以下子系统组成：

| 子系统 | 说明 |
|--------|------|
| 租户管理系统 | 多租户架构下的租户管理、配置、统计 |
| 项目管理系统 | 审核项目的创建、配置、管理 |
| 规则引擎系统 | 审核规则的配置、执行、结果展示 |
| 任务审核系统 | 人工审核任务的分配、执行、统计 |
| 三方接口系统 | 第三方数据源对接、日志、统计 |
| 数据统计系统 | 多维度数据统计与分析报表 |
| 账户权限系统 | 用户账户、角色、权限管理 |
| 系统版本管理 | 系统版本查看、升级、回滚 |
| 备份管理系统 | 备份规则配置、手动/自动备份、备份记录 |

### 2.3 部署模式

系统支持两种部署模式，满足不同客户的需求：

#### 2.3.1 SaaS 多租户模式

**适用场景**：中小型企业、创业公司

| 特性 | 说明 |
|------|------|
| 架构 | 共享资源，平台统一托管 |
| 数据隔离 | 数据库行级隔离 + 租户ID |
| 系统升级 | 平台统一升级，租户无感知 |
| 运维成本 | 低（平台方负责） |
| 定制化 | 有限（标准功能） |
| 费用 | 按月/年订阅 |

#### 2.3.2 独立部署模式

**适用场景**：大型企业、政府机构、金融机构

| 特性 | 说明 |
|------|------|
| 架构 | 独立资源，专属服务器 |
| 数据隔离 | 独立数据库，完全隔离 |
| 系统升级 | 租户自主升级，可选自动/手动 |
| 运维成本 | 高（客户自有运维团队） |
| 定制化 | 完全定制化能力 |
| 费用 | 一次性授权 + 维保费用 |

#### 独立部署环境要求

| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 4 核 | 8 核 |
| 内存 | 8 GB | 16 GB |
| 系统盘 | 100 GB SSD | 200 GB SSD |
| 数据盘 | 500 GB | 1 TB SSD |
| 数据库 | PostgreSQL 15 | PostgreSQL 15 主从 |
| 缓存 | Redis 7 单机 | Redis 7 哨兵 |
| 备份存储 | 本地 | S3/OSS |

### 4.3 规则引擎系统

#### 4.3.1 规则配置

**功能描述**：配置数据审核规则，支持复杂条件组合。

**规则结构**：

```

├── 规则名称
├── 规则描述

│   └── 条件项 (Condition)
│       ├── 字段
│       ├── 运算符
│       └── 比较值

**条件运算符**：

| 运算符 | 说明 | 适用类型 |
|--------|------|----------|
| equals | 等于 | 全部 |
| notEquals | 不等于 | 全部 |
| contains | 包含 | string |
| notContains | 不包含 | string |
| startsWith | 开头是 | string |
| endsWith | 结尾是 | string |
| greaterThan | 大于 | number/date |
| lessThan | 小于 | number/date |
| between | 区间 | number/date |
| in | 属于 | 全部 |
| notIn | 不属于 | 全部 |
| isEmpty | 为空 | 全部 |
| isNotEmpty | 不为空 | 全部 |
| matches | 正则匹配 | string |
| idCardValid | 身份证验证 | string |
| educationVerify | 学历验证 | string |
| degreeVerify | 学位验证 | string |

**条件组嵌套**：

```

#### 4.4.5 二次校验功能

**功能描述**：对规则筛选出的未通过数据进行人工复核。

**触发条件**：
- 规则执行结果为"未通过"
- 配置项开启"需要人工校验"
- 数据被标记为"需要二次校验"

**校验流程**：
1. 审核员执行初审
2. 对未通过数据标记"需要二次校验"
3. 二次校验队列接收任务
4. 另一名审核员或指定人员进行复核
5. 复核结果标记为"二次校验通过"/"二次校验拒绝"
6. 记录二次校验审核员和时间

**二次校验标识**：
- 列表中显示特殊的二次校验标记
- 审核详情页显示"二次校验"标签
- 统计中单独统计二次校验数量

## 6. 系统版本管理（独立部署）

### 6.1 版本信息管理

**功能描述**：管理租户系统的版本信息，支持版本升级。

**版本信息字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 当前版本号 | 字符串 | 如 1.2.3 |
| 版本代码 | 数字 | 用于比较，如 123 |
| 发布时间 | 日期时间 | 版本发布时间 |
| 最后检查时间 | 日期时间 | 最后检查新版本时间 |
| 最后升级时间 | 日期时间 | 最后升级时间 |
| 自动更新状态 | 枚举 | 开启/关闭 |

### 6.2 版本检测

**功能描述**：检测是否有新版本可用。

**检测触发**：
- 租户登录时自动检测
- 管理员手动点击"检查更新"
- 定时自动检测（可配置）

**检测结果显示**：

| 情况 | 显示内容 |
|------|----------|
| 无更新 | "当前已是最新版本" |
| 有可用更新 | "发现新版本 v1.2.3，是否立即更新？" |
| 强制更新 | "系统检测到新版本 v1.2.3，必须更新后才能继续使用" |

### 6.3 版本升级

**功能描述**：执行系统版本升级。

**升级流程**：
1. 管理员触发升级
2. 系统自动创建备份（可选）
3. 下载新版本安装包
4. 执行升级脚本
5. 执行数据库迁移
6. 清理旧文件
7. 升级完成

**升级选项**：

| 选项 | 说明 |
|------|------|
| 升级前自动备份 | 开启后，升级前自动创建全量备份 |
| 强制更新 | 不可跳过，必须升级 |
| 静默升级 | 无需确认，直接升级 |

### 6.4 升级记录

**功能描述**：记录所有版本升级历史。

**记录字段**：

| 字段 | 说明 |
|------|------|
| 升级ID | 唯一标识 |
| 从版本 | 升级前版本 |
| 到版本 | 升级后版本 |
| 升级状态 | 成功/失败/回滚 |
| 开始时间 | 升级开始时间 |
| 完成时间 | 升级完成时间 |
| 备份记录 | 关联的备份记录 |
| 错误信息 | 失败时的错误详情 |
| 操作人 | 执行升级的管理员 |

### 6.5 权限控制

| 权限编码 | 权限名称 | 说明 |
|----------|----------|------|
| SYSTEM_VERSION_VIEW | 查看版本信息 | 查看当前版本 |
| SYSTEM_VERSION_CHECK | 检查更新 | 检测新版本 |
| SYSTEM_UPGRADE | 执行升级 | 执行版本升级 |
| SYSTEM_ROLLBACK | 回滚版本 | 回滚到上一版本 |

## 7. 备份管理系统（独立部署）

### 7.1 备份规则配置

**功能描述**：配置自动备份规则。

**备份类型**：

| 类型 | 说明 | 执行频率 | 保留策略 |
|------|------|----------|----------|
| 全量备份 | 完整数据库备份 | 每周一次 | 保留 30 天 |
| 差异备份 | 与上次全量备份的差异 | 每天一次 | 保留 7 天 |
| 增量备份 | 与上次备份的差异 | 每小时一次 | 保留 3 天 |

**规则配置字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 规则名称 | 字符串 | 规则名称 |
| 备份类型 | 枚举 | 全量/差异/增量 |
| 执行时间 | Cron表达式 | 执行时间，如 "0 2 * * *" |
| 保留天数 | 数字 | 备份保留天数 |
| 存储位置 | 枚举 | 本地/S3/OSS |
| 存储配置 | JSON | 存储连接配置 |
| 状态 | 枚举 | 启用/禁用 |
| 上次执行 | 日期时间 | 上次备份时间 |
| 下次执行 | 日期时间 | 下次备份时间 |

### 7.2 备份执行

**功能描述**：执行数据库备份。

**执行方式**：
- 定时任务自动执行
- 管理员手动触发

**备份流程**：
1. 创建备份记录（状态：待执行）
2. 执行 PostgreSQL 备份
3. 压缩备份文件
4. 计算文件校验码
5. 上传到存储
6. 更新备份记录（状态：成功）
7. 清理过期备份

### 7.3 备份记录

**功能描述**：查看备份执行历史。

**记录字段**：

| 字段 | 说明 |
|------|------|
| 备份ID | 唯一标识 |
| 关联规则 | 所属备份规则 |
| 备份类型 | 全量/差异/增量 |
| 状态 | 待执行/执行中/成功/失败 |
| 开始时间 | 开始执行时间 |
| 完成时间 | 完成时间 |
| 文件路径 | 备份文件存储路径 |
| 文件大小 | 备份文件大小 |
| 校验码 | 文件 SHA256 |
| 错误信息 | 失败时的错误详情 |

**状态筛选**：
- 全部
- 成功
- 失败
- 进行中

### 7.4 备份恢复

**功能描述**：从备份恢复数据。

**恢复流程**：
1. 选择要恢复的备份
2. 确认恢复（将覆盖当前数据）
3. 创建恢复前备份
4. 执行数据恢复
5. 验证数据完整性
6. 恢复完成

### 7.5 权限控制

| 权限编码 | 权限名称 | 说明 |
|----------|----------|------|
| BACKUP_RULE_VIEW | 查看备份规则 | 查看备份规则 |
| BACKUP_RULE_CREATE | 创建备份规则 | 创建备份规则 |
| BACKUP_RULE_EDIT | 编辑备份规则 | 修改备份规则 |
| BACKUP_RULE_DELETE | 删除备份规则 | 删除备份规则 |
| BACKUP_EXECUTE | 执行备份 | 手动执行备份 |
| BACKUP_RESTORE | 恢复备份 | 从备份恢复数据 |
| BACKUP_VIEW | 查看备份记录 | 查看备份历史 |
