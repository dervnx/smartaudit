# 产品概述

## 1. 产品简介

智核 SmartAudit 是一款面向企业级用户的智能审核系统，支持灵活配置审核规则，对接第三方权威数据源，实现自动化审核与人工复核相结合的高效工作模式。

**核心价值**：
- 提升审核效率
- 降低人工误差
- 追溯审核轨迹
- 统计工作绩效

**核心应用场景**：教育机构人事审核、HR人力资源校验、金融机构资质审查、政府部门数据核查等。

## 2. 产品定位

| 维度 | 说明 |
|------|------|
| 目标用户 | 需要对学生信息、职业资格、身份认证、学历学位等数据进行审核校验的企业和机构 |
| 核心功能 | 规则配置 + 自动化审核 + 人工复核 |
| 竞争优势 | 灵活可配置、支持多终端、数据可追溯 |

## 3. 双系统架构

### 3.1 SaaS 管理平台

面向平台运营方，提供：

| 模块 | 功能 |
|------|------|
| 租户管理 | 租户开通、配置、禁用/启用 |
| 版本管理 | 系统版本发布、版本配置（强制更新） |
| 升级管理 | 租户升级监控、升级记录 |
| 全局配置 | 平台级配置、系统参数 |
| 运营统计 | 租户数量、使用情况、收入统计 |

### 3.2 租户管理系统

面向各租户企业，提供：

| 模块 | 功能 |
|------|------|
| 账户权限 | 用户管理、角色权限配置 |
| 项目管理 | 审核项目创建、配置 |
| 规则引擎 | 审核规则可视化配置 |
| 任务审核 | 人工审核任务分配与执行 |
| 三方接口 | 第三方数据源对接配置 |
| 数据统计 | 多维度数据统计报表 |
| 备份管理 | 备份规则、备份记录、恢复 |
| 版本管理 | 版本查看、升级（独立部署） |

## 4. 部署模式

### 4.1 SaaS 多租户模式

```
┌─────────────────────────────────────────────────────┐
│              平台方服务器                             │
│  ┌───────────────────────────────────────────────┐  │
│  │              SaaS 管理平台                       │  │
│  │  (租户开通 / 版本管理 / 运营统计)                │  │
│  └───────────────────────────────────────────────┘  │
│                         │                            │
│         ┌───────────────┼───────────────┐          │
│         ▼               ▼               ▼          │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐   │
│  │   租户 A   │  │   租户 B   │  │   租户 C   │   │
│  │  (共享实例) │  │  (共享实例) │  │  (共享实例) │   │
│  └────────────┘  └────────────┘  └────────────┘   │
└─────────────────────────────────────────────────────┘
```

**特点**：
- 共享资源，成本低
- 平台统一维护
- 数据行级隔离

### 4.2 独立部署模式

```
┌─────────────────────────────────────────────────────┐
│              租户自有服务器                            │
│  ┌───────────────────────────────────────────────┐  │
│  │              租户管理系统                       │  │
│  │  (项目 / 规则 / 审核 / 统计 / 备份)             │  │
│  │  + 系统版本管理                                 │  │
│  └───────────────────────────────────────────────┘  │
│                         │                            │
│         ┌───────────────┼───────────────┐          │
│         ▼               ▼               ▼          │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐   │
│  │ PostgreSQL │  │    Redis   │  │    S3/OSS  │   │
│  │  (独立)    │  │   (独立)   │  │   (备份)   │   │
│  └────────────┘  └────────────┘  └────────────┘   │
└─────────────────────────────────────────────────────┘
```

**特点**：
- 独立资源，隔离性强
- 租户自主运维
- 可定制化开发

## 5. 多端适配

| 端 | 说明 | 适用场景 |
|----|------|----------|
| PC Web | 完整功能 | 管理员、审核员日常办公 |
| Mobile H5 | 核心审核功能 | 移动办公、外出审核 |
| 微信小程序 | 轻量审核 | 扫码、快速审核 |
| Native App | 完整功能 + 离线 | 高频审核用户 |

## 6. 系统角色

| 角色 | 所属 | 说明 |
|------|------|------|
| 平台超级管理员 | SaaS | 平台最高权限，不可删除 |
| 租户最高管理员 | 租户 | 租户创始人，拥有全部权限 |
| 普通管理员 | 租户 | 项目管理、规则配置 |
| 审核员 | 租户 | 执行审核任务 |
| 普通员工 | 租户 | 查看数据 |

## 7. 核心功能模块

### 7.1 规则引擎

```
┌─────────────────────────────────────────────────────┐
│                    规则配置                            │
├─────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────┐    │
│  │ 条件组 A (全部满足 ALL)                      │    │
│  │ ├── 条件1: 年龄 >= 18                       │    │
│  │ ├── 条件2: 学历 in [本科,硕士,博士]          │    │
│  │ └── 条件组 B (任一满足 ANY)                  │    │
│  │     ├── 条件3: 驾照类型 = A                   │    │
│  │     └── 条件4: 驾照类型 = B                   │    │
│  └─────────────────────────────────────────────┘    │
│                                                     │
│  支持运算符: equals, contains, between, regex,      │
│            idCardValid, educationVerify...         │
└─────────────────────────────────────────────────────┘
```

### 7.2 任务审核

```
┌─────────────────────────────────────────────────────┐
│                   任务审核流程                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  数据提交 ──► 规则筛选 ──► 待审核队列                │
│                               │                      │
│                               ▼                      │
│                    ┌──────────────────┐            │
│                    │   任务分配        │            │
│                    │ (轮询/负载/效率)  │            │
│                    └────────┬─────────┘            │
│                             │                       │
│                             ▼                       │
│                    ┌──────────────────┐            │
│                    │   审核员工作台    │            │
│                    │   (FIFO 顺序)    │            │
│                    └────────┬─────────┘            │
│                             │                       │
│           ┌─────────────────┼─────────────────┐    │
│           ▼                 ▼                 ▼    │
│      [通过]            [拒绝]           [二次校验]  │
│                                                     │
└─────────────────────────────────────────────────────┘
```
