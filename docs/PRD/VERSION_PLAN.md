## 8. 版本兼容性

### 8.1 数据库迁移

- 每个版本提供数据库迁移脚本
- 支持从上一版本平滑升级
- 不支持跨版本升级（如从V1.0直接升级到V1.2）

### 8.2 API 兼容性

- 同一大版本内API保持兼容
- breaking changes会提前一个版本通知
- API版本号：/api/v1/, /api/v2/

### 8.3 前端兼容性

- 同一大版本内前端保持兼容
- 建议用户使用最新版本浏览器

## 9. 版本号规范

### 9.1 版本格式

```
主版本号.次版本号.修订版本号
  Major     Minor     Patch
    1        2         3
```

### 9.2 版本递增规则

| 类型 | 触发条件 | 示例 |
|------|----------|------|
| 主版本号 | 不兼容的架构变更 | 1.x.x → 2.0.0 |
| 次版本号 | 向下兼容的新功能 | x.1.x → x.2.0 |
| 修订版本号 | 向下兼容的问题修复 | x.x.1 → x.x.2 |

### 9.3 version_code 计算

```
version_code = Major * 10000 + Minor * 100 + Patch

例如：
  1.2.3 => 10203
  2.0.0 => 20000
```

## 11. 维护周期

| 版本 | 维护状态 | 说明 |
|------|----------|------|
| 当前稳定版 | 主动维护 | 修复bug和安全问题 |
