# 租户后台需求文档

## 1. 系统概述

租户管理系统是各租户企业使用的管理系统，用于管理企业内部的用户、项目、规则、审核任务等。

**使用对象**：租户管理员、审核员、普通员工

#### 3.2.2 创建项目

**页面**：/tenant/projects/create

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 项目名称 | 字符串 | ✓ | 2-100字符 |
| 项目代码 | 字符串 | ✓ | 小写字母+数字，3-30字符 |
| 项目简介 | 文本 | ✗ | 项目描述 |
| 入参数据项 | JSON | ✓ | 数据项定义 |
| 审核规则 | 多选 | ✓ | 关联的规则 |
| 任务分配策略 | 选择 | ✓ | 分配方式 |
| 审核模式配置 | 对象 | ✓ | 审核流程配置（见下方） |
| 状态 | 枚举 | ✓ | active/inactive |

**审核模式配置**：

| 模式 | 代码 | 说明 |
|------|------|------|
| 自动审核 | auto | 规则通过即终态通过，规则拒绝即终态拒绝，无需人工 |
| 人工审核 | manual | 所有数据都需审核员复核 |
| 混合模式 | mixed | 规则通过自动终态，规则拒绝分配审核员 |

| 配置项 | 说明 |
|--------|------|
| 是否需要人工复核 | 是否启用人工审核流程 |
| 规则通过时自动终态 | 规则通过时无需审核员，直接通过 |
| 是否启用二次复核 | 对已审核数据可再次审核 |
| 仅对拒绝数据启用复核 | 只有审核拒绝时才可二次复核 |

### 3.4 任务审核系统

#### 3.4.1 任务分配策略

**页面**：/tenant/tasks/config

**策略类型**：

| 策略 | 代码 | 说明 |
|------|------|------|
| 轮询分配 | round_robin | 按顺序轮流分配 |
| 负载均衡 | load_balance | 分配给任务最少的人 |
| 效率优先 | efficiency | 分配给完成最快的 |
| 随机分配 | random | 随机分配 |
| 手动分配 | manual | 手动指定 |

**配置示例**：

```json
{
  "strategy": "load_balance",
  "rules": [
    {"type": "max_per_auditor", "value": 100},
    {"type": "min_tasks_threshold", "value": 10}
  ]
}
```
